# Phase 10 — Portfolio, Specialization & Getting Hired

**Goal:** Turn your skills into a job. Build a portfolio, pick a specialization, and prepare for interviews.
**Time:** Ongoing — start during Phase 6, intensify after Phase 9.

---

## 🎯 Outcomes
You have a public portfolio, a clear specialization, an optimized resume/profile, and you can pass technical interviews.

## ✅ Checklist

### Portfolio (your strongest signal)
- [ ] 3–5 polished projects on GitHub, each with a clear README + demo
- [ ] At least one **deployed** project with a live URL
- [ ] At least one project that solves a *real* problem (yours or someone's)
- [ ] Clean commit history; pinned repos on your profile
- [ ] Write up 2–3 projects as blog posts / LinkedIn posts
- [ ] (Bonus) Contribute to an open-source AI project

### Pick a specialization
- [ ] **LLM / GenAI Engineer** — RAG, agents, LLM apps (hottest, fastest hire)
- [ ] **ML Engineer** — training, deployment, MLOps
- [ ] **Data Scientist** — analysis, experimentation, modeling
- [ ] **MLOps / Platform Engineer** — infra, scaling, reliability
- [ ] **Computer Vision** / **NLP** / **Recommender Systems** (domain depth)
- [ ] **Research Engineer** — papers, novel models (needs more math)

### Profile & applications
- [ ] Resume tuned to ML/AI (projects > coursework, show impact + metrics)
- [ ] LinkedIn & GitHub aligned with target role
- [ ] Network: AI communities, meetups, Twitter/X, Discord
- [ ] Track applications; apply to roles slightly above your level

### Keep learning (the field moves fast)
- [ ] Follow papers ([arXiv](https://arxiv.org/list/cs.LG/recent), [Papers with Code](https://paperswithcode.com/))
- [ ] Newsletters/podcasts (The Batch, Latent Space, Import AI)
- [ ] Reproduce one new paper or model per quarter

## 🎤 Interview preparation
See the dedicated guide: **[interview/PREP.md](../interview/PREP.md)** — covers ML concepts, coding, ML system design, take-homes, and behavioral.

## 📚 Best resources
- **Interview** — [*Machine Learning Interviews* by Chip Huyen (free book)](https://huyenchip.com/ml-interviews-book/) · [*Designing ML Systems*](https://www.oreilly.com/library/view/designing-machine-learning/9781098107956/)
- **Portfolio inspiration** — browse top AI engineers' GitHub profiles
- **Practice** — [LeetCode](https://leetcode.com/) (some companies still test DS&A) · mock interviews

## 🛠️ Capstone project
**Build one impressive, end-to-end, deployed AI product** that combines everything: a real problem, a real model or LLM system, RAG or an agent, a clean UI, deployed to the cloud, monitored, with a great README and a write-up. This is the centerpiece of your portfolio.

🎉 **You made it.** Now go build things and ship them. The field rewards builders.
