# 🛠️ Portfolio Projects

Projects teach more than any course. Build these, push each to GitHub with a clear README, and deploy the ⭐ ones. Ordered by difficulty.

> **Rule of thumb:** a portfolio of 4–5 *finished, deployed* projects beats 20 half-done tutorials.

## 🟢 Beginner (Phases 0–3)
- **CSV data tool** — CLI that cleans a CSV and outputs a JSON report.
- **EDA notebook** — full exploratory analysis of a Kaggle dataset with insights + charts.
- **Kaggle Titanic / House Prices** — end-to-end classical ML with feature engineering + tuning.
- **Spam / sentiment classifier** — TF-IDF + logistic regression baseline.

## 🟡 Intermediate (Phases 4–6)
- **Image classifier** — CNN from scratch, then beat it with transfer learning (ResNet). ⭐
- **micrograd reimplementation** — tiny autograd engine (cements backprop).
- **Fine-tuned transformer** — DistilBERT fine-tuned on your own text-classification task.
- **LLM app** — document summarizer / structured extractor / code reviewer with evals. ⭐ deploy it.

## 🔴 Advanced (Phases 7–9)
- **Chat with your docs (RAG)** — chunking + embeddings + vector DB + cited answers + hybrid search & re-ranking. ⭐ deploy it.
- **AI agent** — research/coding/support agent with tool use, loop control, and a trace/eval. ⭐
- **Productionized service** — any above project Dockerized, behind FastAPI, deployed, monitored, with CI. ⭐⭐

## 🏆 Capstone (Phase 10)
- **End-to-end AI product** — real problem + RAG/agent + UI + cloud deploy + monitoring + write-up. The centerpiece of your portfolio. ⭐⭐⭐

### Project README checklist (do this for every project)
- [ ] One-line description + screenshot/GIF or demo link
- [ ] Problem it solves & why
- [ ] Tech stack & architecture diagram
- [ ] How to run it locally
- [ ] What you learned / trade-offs made
- [ ] Live demo URL (if deployed)
